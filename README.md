A full-fledged, modular, and industry-grade Machine Learning pipeline that recommends movies using Content-Based Filtering (TF-IDF & Cosine Similarity). Designed from scratch using clean architecture principles, dynamic configuration mapping, and separated ML component lifecycles.

---

## What It Does 💡

This platform acts as an intelligent movie discovery engine. When you search for a movie:
1. It queries our **FastAPI Backend** and seamlessly connects with the **TMDB API** to fetch rich real-time metadata (posters, genres, overviews).
2. It queries our offline **Scipy TF-IDF Machine Learning model** matrix to find mathematically similar movies based on their combined textual features (genres, overviews, and taglines).
3. The results are beautifully rendered via a fast, responsive **Streamlit UI**, providing "More Like This" recommendations intuitively.

## How It Helps 🚀

Traditional python scripts (like Jupyter Notebooks) are challenging to scale, test, or deploy. This implementation solves the "Notebook-to-Production" headache.

- **For Users**: Provides highly accurate, instantly accessible movie recommendations with a gorgeous visual grid.
- **For Evaluators/Engineers**: Showcases how a barebones ML idea is structured into an enterprise-ready pipeline with explicit separation of components (`Ingestion` -> `Transformation` -> `Trainer` -> `Evaluation`), type-safe data validation using `dataclasses`, scalable exception handling, centralized custom logging, and decoupled configuration files (`yaml`).

## Dataset & Features 📊

The model is trained on the popular `movies_metadata.csv` containing thousands of film entries.

### Feature Engineering
The pipeline uses **Content-Based Filtering**. It synthesizes strings together to create a deep semantic representation of a movie:
- **Genres** (Action, Adventure, Sci-Fi)
- **Overview** (Plot summaries)
- **Taglines** (Marketing slogans)

These text blobs are processed through a **Term Frequency-Inverse Document Frequency (TF-IDF)** Vectorizer yielding a sparse matrix, which generates semantic similarity scores across the whole dataset instantly via matrix dot products (`Cosine Similarity`).

## Project Architecture 🏗️

The codebase follows the strict ML Component lifecycle architecture driven by `config.yaml`. No hardcoded configurations exist in the pipeline.

```text
movie-rec/
│
├── src/
│   ├── components/
│   │   ├── data_ingestion.py        # Fetches/copies dataset into artifacts
│   │   ├── data_transformation.py   # Cleans data, creates combined text features
│   │   ├── model_trainer.py         # Trains TF-IDF Vectorizer & creates sparse similarity matrix
│   │   ├── model_evaluation.py      # Performs validation on matrix shape & sparsity
│   │
│   ├── pipeline/
│   │   ├── training_pipeline.py     # Orchestrates component lifecycle automatically
│   │   ├── prediction_pipeline.py   # Loads generated pickle models for quick backend inference
│   │
│   ├── config/
│   │   ├── configuration.py         # Reads config.yaml -> Returns typed Entities
│   │
│   ├── entity/
│   │   ├── config_entity.py         # Dataclass wrappers for configurations
│   │   ├── artifact_entity.py       # Dataclass wrappers for generated I/O paths
│   │
│   ├── utils/
│   │   ├── common.py                # Reusable loaders, savers, and YAML parsers
│   │
│   ├── logger.py                    # Centralized rotating custom logging module
│   ├── exception.py                 # Custom exception class for detailed traceback propagation
│
├── artifacts/                       # Auto-generated by pipeline
│   ├── data_ingestion/
│   ├── data_transformation/
│   ├── model_trainer/
│   ├── logs/
│
├── config/
│   ├── config.yaml                  # Application wide master configurations
│
├── app.py                           # Streamlit UI visual frontend
├── api.py                           # FastAPI Backend integration with Prediction Pipeline

├── requirements.txt
├── setup.py                         # Allows package importing (-e .)
```

## How To Run 💻

### 1. Installation

First, clone the repository and install the production requirements. This will securely install the `src` folder as a local python package.

```bash
pip install -r requirements.txt
```

Set up your `.env` file in the root directory:
```env
TMDB_API_KEY=your_api_key_here
```

### 2. Train the Machine Learning Pipeline

Before making predictions, the ML models (`pkl` format) need to be compiled. `main.py` orchestrates the entire lifecycle automatically.

```bash
python main.py
```

*Expected Result: You will observe logs documenting the inception of Data Ingestion up to Model Evaluation. The `./artifacts` folder will populate with the vectorized `tfidf_matrix.pkl` models.*

### 3. Start the Application Backend (FastAPI)

Launch the robust inference REST backend that consumes the newly generated artifacts.

```bash
uvicorn api:app --reload --port 8000
```

### 4. Start the Frontend (Streamlit)

In a new terminal, spin up the beautiful user interface to browse recommendations seamlessly.

```bash
streamlit run app.py
```
